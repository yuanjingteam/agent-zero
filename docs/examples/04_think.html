04 - 思考：推理与问题解决

难度：中级 时长：~20 分钟 关键概念：推理、问题分解、局限性

概述

本示例演示了如何将大语言模型（LLM）配置为推理智能体（Reasoning Agent），使其具备分析思维和定量问题求解能力。它展示了从简单文本生成到复杂认知任务之间的桥梁。

什么是推理智能体？

推理智能体是一种通过精心设计的系统提示词（System Prompt）配置的 LLM，能够执行逻辑分析、数学计算和多步问题求解。

类比理解

普通对话 推理智能体 ───────────── ────────────────── "你能帮我吗？" "我是一名数学家。 "当然！你需要什么？" 我系统地分析问题 并计算精确答案。"

推理的挑战

为什么推理对 LLM 来说很难

┌───────────────────────────────────────┐ │ LLM 训练 │ │ "预测文本中的下一个词" │ │ │ │ 并非显式训练以下能力： │ │ • 逐步逻辑推理 │ │ • 算术计算 │ │ • 追踪多个变量 │ │ • 系统性问题分解 │ └───────────────────────────────────────┘

然而，它们可以从训练数据中学习推理模式，并通过系统提示词加以引导。

通过系统提示词实现推理

配置模式

┌─────────────────────────────────────────┐ │ 系统提示词组件 │ ├─────────────────────────────────────────┤ │ 1. 角色："专业推理者" │ │ 2. 任务："分析并解决问题" │ │ 3. 方法："计算精确答案" │ │ 4. 输出："单个数值" │ └─────────────────────────────────────────┘ ↓ 推理行为

推理任务类型

定量推理（本示例）： 问题 → 统计实体 → 计算 → 单位换算 → 答案 逻辑推理： 前提 → 应用规则 → 推导结论 → 答案 分析推理： 数据 → 识别模式 → 形成假设 → 得出结论

LLM 如何"推理"

模式匹配 vs 真正的推理

┌─────────────────────────────────────────────┐ │ LLM 实际的工作方式 │ │ │ │ 1. 模式识别 │ │ "这看起来是一个计数问题" │ │ │ │ 2. 模板应用 │ │ "类似的问题遵循这种模式" │ │ │ │ 3. 统计推断 │ │ "这些数字可能以这种方式组合" │ │ │ │ 4. 学到的程序 │ │ "我见过这种类型的计算" │ └─────────────────────────────────────────────┘

推理过程

输入：复杂应用题 ↓ ┌────────────┐ │ 解析 │ 识别实体和关系 └────────────┘ ↓ ┌────────────┐ │ 分解 │ 拆分为子问题 └────────────┘ ↓ ┌────────────┐ │ 计算 │ 执行算术运算 └────────────┘ ↓ ┌────────────┐ │ 综合 │ 合并结果 └────────────┘ ↓ 最终答案

问题复杂度层级

简单 困难 │ │ │ 简单 多步 嵌套 隐式 │ │ 算术 逻辑 条件 推理 │ │ │ └─────────────────────────────────────────────┘ 示例： 简单： "5 + 3 等于多少？" 中等： "如果 3 个苹果每个 2 美元，总共多少钱？" 困难： "统计具有复杂家庭关系的家庭成员人数"

本示例的复杂度

土豆问题极其复杂：

  • 多个实体（15 人以上）
  • 关系推理（家谱）
  • 条件逻辑（如果已婚则...）
  • 否定条件（已故人员）
  • 特殊情况（饮食限制）
  • 多次计算
  • 单位换算

纯 LLM 推理的局限性

常见失败模式

  • 计数错误：统计 15 个具有复杂关系的人时，可能输出 "14" 或 "16"
  • 算术错误：在多步计算的中间步骤出错
  • 上下文丢失：忘记前面的信息

改进推理：演进路径

级别 1：纯提示（本示例）

用户 → LLM → 答案 ↑ 系统提示词

局限性：所有推理都在 LLM 内部完成，没有验证，没有工具，过程不可见。

级别 2：思维链（Chain-of-Thought）

用户 → LLM → 展示过程 → 答案 ↑ "解释你的推理"

级别 3：工具增强（simple-agent）

用户 → LLM ⟷ 工具 → 答案 ↑ （计算器） 系统提示词

级别 4：ReAct 模式（react-agent）

用户 → LLM → 思考 → 行动 → 观察 ↑ ↓ ↓ ↓ 系统 推理 工具 结果 提示词 使用 ↑ ↓ ↓ └───────────迭代────┘

关键要点

  1. 系统提示词赋能推理：合理的配置可以将 LLM 转变为推理智能体
  2. 局限性存在：纯 LLM 推理在复杂问题上容易出错
  3. 工具有帮助：外部计算（计算器等）可以提高准确性
  4. 迭代很重要：多步推理模式（如 ReAct）效果更好
  5. 透明性有价值：可见的推理过程有助于调试和验证
