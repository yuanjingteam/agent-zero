03 - 翻译：系统提示与智能体专业化

难度：入门 时长：~15 分钟 关键概念：系统提示、提示工程、行为约束

概述

本示例演示如何使用系统提示词（System Prompts）将通用的 LLM（大语言模型）转变为专门化智能体（Specialized Agent）。关键洞察在于：你不需要为不同任务准备不同的模型——你需要的是不同的指令。

什么是系统提示词？

系统提示词是一条持久性指令，用于在整个对话会话中塑造 AI 的行为。

类比

没有系统提示词 有系统提示词 ───────────────────── ────────────────────── "嗨，我是 AI。" "我是一名专业翻译， 精通科学德语。 "我能为你做什么？" 我遵循严格的质量准则 和输出格式。"

系统提示词的工作原理

上下文结构

┌─────────────────────────────────────────────┐ │ 上下文窗口（Context Window） │ │ │ │ ┌───────────────────────────────────────┐ │ │ │ 系统提示词（始终存在） │ │ │ │ "你是一名专业翻译……" │ │ │ │ "遵循以下规则……" │ │ │ └───────────────────────────────────────┘ │ │ ↓ │ │ ┌───────────────────────────────────────┐ │ │ │ 用户消息 │ │ │ │ "翻译这段文本……" │ │ │ └───────────────────────────────────────┘ │ │ ↓ │ │ ┌───────────────────────────────────────┐ │ │ │ AI 回复 │ │ │ │ （受系统提示词塑造） │ │ │ └───────────────────────────────────────┘ │ └─────────────────────────────────────────────┘

系统提示词位于上下文的最顶部，影响每一次回复。

智能体专门化模式

转变流程

┌──────────────────┐ ┌─────────────────┐ ┌──────────────────┐ │ 通用模型 │ + │ 系统提示词 │ = │ 专门化智能体 │ │ │ │ │ │ │ │ • 了解很多知识 │ │ • 定义角色 │ │ • 翻译智能体 │ │ • 没有特定角色 │ │ • 设定规则 │ │ • 编码智能体 │ │ │ │ • 约束输出 │ │ • 分析智能体 │ └──────────────────┘ └─────────────────┘ └──────────────────┘

专门化示例

// 翻译智能体（本示例）
systemPrompt = 角色 + 规则 + 输出格式

// 代码助手
systemPrompt: "You are an expert programmer.
Always provide working code with comments.
Explain complex logic."

// 数据分析员
systemPrompt: "You are a data analyst.
Always show your calculations step-by-step.
Cite data sources when available."

有效系统提示词的解剖

五大组成部分

┌─────────────────────────────────────────┐ │ 1. 角色定义（Role Definition） │ │ "你是一名 [具体角色]……" │ ├─────────────────────────────────────────┤ │ 2. 任务描述（Task Description） │ │ "你的目标是……" │ ├─────────────────────────────────────────┤ │ 3. 行为规则（Behavioral Rules） │ │ "始终做 X，绝不做 Y……" │ ├─────────────────────────────────────────┤ │ 4. 输出格式（Output Format） │ │ "按以下格式输出你的回复……" │ ├─────────────────────────────────────────┤ │ 5. 约束条件（Constraints） │ │ "不要包含……" │ └─────────────────────────────────────────┘

本示例的结构

角色： "专业科学翻译" 任务： "将英语精确翻译为德语" 规则： 8 条具体翻译指南 格式： 地道德语，科学风格 约束： "仅输出翻译文本，不加解释"

为什么详细的系统提示词很重要

质量影响

详细程度输出质量
非常简略不可预测
基本角色有一定一致性
详细说明高度一致
过度详细可能令模型困惑

系统提示词设计模式

模式 1：角色扮演

"你是一名 [职业]，在 [领域] 方面具有专业知识……"

让模型采用该角色视角。

模式 2：基于规则

"遵循以下规则：
1. 始终……
2. 绝不……
3. 当 X 发生时，做 Y……"

明确的约束带来可预测的行为。

模式 3：输出格式化

"按以下格式输出你的回复：
- JSON
- Markdown
- 纯文本
- 逐步列表"

模式 4：上下文感知

"你记得：[先前的事实]
你知道：[领域知识]
当前情况：[上下文]"

聊天包装器详解

不同的模型需要不同的对话格式：

模型类型所需格式包装器
Llama 2/3Llama 格式LlamaChatWrapper
GPT 风格ChatML 格式ChatMLWrapper
Harmony 模型Harmony 格式HarmonyChatWrapper
你的消息 → [聊天包装器] → 格式化的提示词 → 模型 ↓ 添加特殊 token： <|system|>, <|user|>, <|assistant|>
