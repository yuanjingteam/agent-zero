# 概念：系统提示词与智能体专门化

## 概述

本示例演示如何使用**系统提示词（System Prompts）**将通用的 LLM（大语言模型）转变为**专门化智能体（Specialized Agent）**。关键洞察在于：你不需要为不同任务准备不同的模型——你需要的是不同的指令。

## 什么是系统提示词？

**系统提示词**是一条持久性指令，用于在整个对话会话中塑造 AI 的行为。

### 类比
想象一下招聘某人来承担一份工作：

```
没有系统提示词                    有系统提示词
─────────────────────         ──────────────────────
"嗨，我是 AI。"                "我是一名专业翻译，
                              精通科学德语。
"我能为你做什么？"             我遵循严格的质量准则
                              和输出格式。"
```

## 系统提示词的工作原理

### 上下文结构

```
┌─────────────────────────────────────────────┐
│           上下文窗口（Context Window）        │
│                                             │
│  ┌───────────────────────────────────────┐ │
│  │  系统提示词（始终存在）                 │ │
│  │  "你是一名专业翻译……"                  │ │
│  │  "遵循以下规则……"                     │ │
│  └───────────────────────────────────────┘ │
│                    ↓                        │
│  ┌───────────────────────────────────────┐ │
│  │  用户消息                             │ │
│  │  "翻译这段文本……"                     │ │
│  └───────────────────────────────────────┘ │
│                    ↓                        │
│  ┌───────────────────────────────────────┐ │
│  │  AI 回复                              │ │
│  │  （受系统提示词塑造）                   │ │
│  └───────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
```

系统提示词位于上下文的最顶部，影响**每一次**回复。

## 智能体专门化模式

### 转变流程

```
┌──────────────────┐    ┌─────────────────┐    ┌──────────────────┐
│   通用模型       │ +  │  系统提示词     │ =  │  专门化智能体    │
│                  │    │                 │    │                  │
│ • 了解很多知识   │    │ • 定义角色      │    │ • 翻译智能体     │
│ • 没有特定角色   │    │ • 设定规则      │    │ • 编码智能体     │
│                  │    │ • 约束输出      │    │ • 分析智能体     │
└──────────────────┘    └─────────────────┘    └──────────────────┘
```

### 专门化示例

**翻译智能体（本示例）：**
```
系统提示词 = 角色 + 规则 + 输出格式
```

**代码助手：**
```javascript
systemPrompt: "You are an expert programmer. 
Always provide working code with comments.
Explain complex logic."
```

**数据分析员：**
```javascript
systemPrompt: "You are a data analyst.
Always show your calculations step-by-step.
Cite data sources when available."
```

## 有效系统提示词的解剖

### 五大组成部分

```
┌─────────────────────────────────────────┐
│  1. 角色定义（Role Definition）          │
│  "你是一名 [具体角色]……"                │
├─────────────────────────────────────────┤
│  2. 任务描述（Task Description）         │
│  "你的目标是……"                         │
├─────────────────────────────────────────┤
│  3. 行为规则（Behavioral Rules）         │
│  "始终做 X，绝不做 Y……"                 │
├─────────────────────────────────────────┤
│  4. 输出格式（Output Format）            │
│  "按以下格式输出你的回复……"              │
├─────────────────────────────────────────┤
│  5. 约束条件（Constraints）              │
│  "不要包含……"                           │
└─────────────────────────────────────────┘
```

### 本示例的结构

```
角色：      "专业科学翻译"
任务：      "将英语精确翻译为德语"
规则：      8 条具体翻译指南
格式：      地道德语，科学风格
约束：      "仅输出翻译文本，不加解释"
```

## 为什么详细的系统提示词很重要

### 对比研究

**简略的系统提示词：**
```javascript
systemPrompt: "Translate to German"
```

**结果：**
- 可能添加不必要的解释
- 术语不一致
- 正式程度混乱
- 多余的对话文本

**详细的系统提示词（本示例）：**
```javascript
systemPrompt: `You are a professional translator...
- Rule 1: Preserve technical accuracy
- Rule 2: Use idiomatic German
- Rule 3: Follow scientific conventions
...
DO NOT add any explanations`
```

**结果：**
- 一致的质量
- 正确的术语
- 规范的格式
- 仅输出翻译内容

### 质量影响

```
详细程度              输出质量
───────────         ─────────────────
非常简略   →        不可预测
基本角色   →        有一定一致性
详细说明   →        高度一致
过度详细   →        可能令模型困惑
```

## 系统提示词设计模式

### 模式 1：角色扮演
```
"你是一名 [职业]，在 [领域] 方面具有专业知识……"
```
让模型采用该角色视角。

### 模式 2：基于规则
```
"遵循以下规则：
1. 始终……
2. 绝不……
3. 当 X 发生时，做 Y……"
```
明确的约束带来可预测的行为。

### 模式 3：输出格式化
```
"按以下格式输出你的回复：
- JSON
- Markdown
- 纯文本
- 逐步列表"
```
控制回复的结构。

### 模式 4：上下文感知
```
"你记得：[先前的事实]
你知道：[领域知识]
当前情况：[上下文]"
```
为模型提供相关信息进行预处理。

## 这与 AI 智能体的关系

### 智能体 = 模型 + 系统提示词 + 工具

```
┌────────────────────────────────────────────┐
│             AI 智能体                       │
│                                            │
│  ┌──────────────────────────────────────┐ │
│  │  系统提示词（智能体的"身份"）         │ │
│  └──────────────────────────────────────┘ │
│                  ↓                         │
│  ┌──────────────────────────────────────┐ │
│  │  LLM（智能体的"大脑"）               │ │
│  └──────────────────────────────────────┘ │
│                  ↓                         │
│  ┌──────────────────────────────────────┐ │
│  │  工具（智能体的"双手"）[可选]        │ │
│  └──────────────────────────────────────┘ │
└────────────────────────────────────────────┘
```

**在本示例中：**
- 系统提示词："你是一名翻译……"
- LLM：Apertus-8B 模型
- 工具：无（翻译由模型本身完成）

**在更复杂的智能体中：**
- 系统提示词："你是一名研究助手……"
- LLM：任意模型
- 工具：网络搜索、计算器、文件访问等

## 实际应用

### 1. 领域专门化
```
医疗 → "你是一名医疗专业人员……"
法律 → "你是一名法律专家……"
技术 → "你是一名工程师……"
```

### 2. 输出控制
```
JSON API → "始终以有效的 JSON 格式回复"
Markdown → "所有回复使用 Markdown 格式"
代码 → "仅输出可执行代码"
```

### 3. 行为约束
```
简洁 → "最多使用 2 个句子"
详细 → "用示例进行透彻说明"
中立 → "避免表达观点，仅陈述事实"
```

### 4. 多语言支持
```javascript
systemPrompt: `You are a multilingual assistant.
Respond in the same language as the input.`
```

## 聊天包装器详解

不同的模型需要不同的对话格式：

```
模型类型           所需格式              包装器
──────────────   ───────────────────   ─────────────────
Llama 2/3        Llama 格式            LlamaChatWrapper
GPT 风格          ChatML 格式           ChatMLWrapper
Harmony 模型      Harmony 格式          HarmonyChatWrapper
```

**它们的作用：**
```
你的消息 → [聊天包装器] → 格式化的提示词 → 模型
                    ↓
          添加特殊 token：
          <|system|>, <|user|>, <|assistant|>
```

聊天包装器确保模型能够理解哪部分是系统提示词、哪部分是用户消息等。

## 核心要点

1. **系统提示词非常强大**：它们从根本上改变了模型的行为
2. **越详细越好**：更具体的指令 = 更一致的结果
3. **结构很重要**：角色 + 规则 + 格式 + 约束
4. **无需重新训练**：同一个模型，不同的行为
5. **智能体的基础**：系统提示词是构建专门化智能体的第一步
