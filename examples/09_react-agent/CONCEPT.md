# 概念：AI 智能体的 ReAct 模式

## 什么是 ReAct？

**ReAct**（Reasoning + Acting，推理与行动）是一个将以下能力结合的框架：
- **推理（Reasoning）**：逐步思考问题
- **行动（Acting）**：使用工具完成子任务
- **观察（Observing）**：从工具结果中学习

这使得智能体能够可靠地解决复杂的多步骤问题。

## 核心模式

```
┌─────────────┐
│    问题     │
└──────┬──────┘
       │
       ▼
┌─────────────────────────────────────┐
│          ReAct 循环                 │
│                                     │
│  ┌──────────────────────────────┐  │
│  │  1. 思考（THOUGHT）          │  │
│  │  "我需要做什么？"            │  │
│  └─────────────┬────────────────┘  │
│                ▼                    │
│  ┌──────────────────────────────┐  │
│  │  2. 行动（ACTION）           │  │
│  │  调用工具并传入参数          │  │
│  └─────────────┬────────────────┘  │
│                ▼                    │
│  ┌──────────────────────────────┐  │
│  │  3. 观察（OBSERVATION）      │  │
│  │  接收工具返回结果            │  │
│  └─────────────┬────────────────┘  │
│                │                    │
│                └──► 重复或          │
│                     最终答案        │
└─────────────────────────────────────┘
```

## 为什么 ReAct 很重要

### 传统大语言模型（LLM）的困难：
1. **复杂计算** — 算术错误
2. **多步骤问题** — 容易丢失进度
3. **使用工具** — 不知道何时使用、如何使用
4. **解释决策** — 黑盒推理

### ReAct 解决了这些问题：
1. **可靠的计算** — 委托给工具执行
2. **结构化的进度** — 显式的步骤
3. **工具编排** — 知道何时使用哪个工具
4. **透明的推理** — 可见的思考过程

## 三大组件

### 1. 思考（Thought，推理）

智能体进行以下推理：
- 需要什么信息
- 使用哪个工具
- 结果是否合理
- 下一步该做什么

示例：
```
Thought: 我需要计算 15 × 8 来得出收入
```

### 2. 行动（Action，工具使用）

智能体使用特定参数调用工具：

示例：
```
Action: multiply(15, 8)
```

### 3. 观察（Observation，学习）

智能体接收并解读工具结果：

示例：
```
Observation: 120
```

## 完整示例

```
问题："如果 15 件商品每件 $8，20 件商品每件 $8，
      总收入是多少？"

Thought: 首先我需要计算 15 件商品的收入
Action: multiply(15, 8)
Observation: 120

Thought: 现在我需要计算 20 件商品的收入
Action: multiply(20, 8)
Observation: 160

Thought: 现在我需要将两项收入相加
Action: add(120, 160)
Observation: 280

Thought: 我已经得到了最终答案
Answer: 总收入为 $280
```

## 核心优势

### 1. 可靠性
- 工具提供准确的结果
- 没有算术错误
- 可验证的计算

### 2. 透明性
- 可以看到每个推理步骤
- 理解决策过程
- 便于调试

### 3. 可扩展性
- 能处理复杂问题
- 分解为可管理的步骤
- 按需添加更多工具

### 4. 灵活性
- 适用于任何工具
- 能适应问题的复杂度
- 需要时可自我纠错

## 与其他方法的比较

### 零样本提示（Zero-Shot Prompting）
```
用户："计算 15×8 + 20×8"
LLM："答案是 279"  ❌ 错误！
```
**问题**：LLM 在脑中计算，容易出错

### 思维链（Chain-of-Thought）
```
用户："计算 15×8 + 20×8"
LLM："让我逐步思考：
     15×8 = 120
     20×8 = 160
     120+160 = 279"  ❌ 还是错了！
```
**问题**：展示了计算过程但仍然算错

### ReAct（本实现）
```
用户："计算 15×8 + 20×8"
智能体：
  Thought: 计算 15×8
  Action: multiply(15, 8)
  Observation: 120
  
  Thought: 计算 20×8
  Action: multiply(20, 8)
  Observation: 160
  
  Thought: 将结果相加
  Action: add(120, 160)
  Observation: 280
  
  Answer: 280  ✅ 正确！
```
**成功**：使用工具，获得准确结果

## 实现策略

### 1. 强制执行显式模式

强制 LLM 遵循结构：
```javascript
systemPrompt: `CRITICAL: Follow this EXACT pattern:
Thought: [reasoning]
Action: [tool call]
Observation: [result]
...
Answer: [final answer]`
```

### 2. 迭代控制

防止无限循环：
```javascript
maxIterations = 10  // 安全限制
```

### 3. 流式输出

实时展示进度：
```javascript
onTextChunk: (chunk) => {
    process.stdout.write(chunk);
}
```

### 4. 答案检测

判断何时停止：
```javascript
if (response.includes("Answer:")) {
    return fullResponse;  // 完成！
}
```

## 局限性与注意事项

### 1. 迭代成本
每次思考/行动/观察循环都会消耗 token 和时间。

**解决方案**：使用高效模型，限制迭代次数

### 2. 工具质量
ReAct 的效果取决于其工具的质量。

**解决方案**：构建健壮、经过充分测试的工具

### 3. 提示词工程
系统提示词必须非常清晰。

**解决方案**：进行充分测试，持续迭代优化提示词

### 4. 错误处理
工具可能会失败或返回意外结果。

**解决方案**：添加错误处理和验证

## 高级模式

### 自我纠错
```
Thought: 这个结果看起来不对
Action: verify(previous_result)
Observation: 检测到错误
Thought: 让我重新计算
Action: multiply(15, 8)  # 重试
```

### 元推理（Meta-Reasoning）
```
Thought: 我已经用了 5 次迭代，应该尽快结束
Action: summarize_progress()
Observation: 仍需将最后的数字相加
Thought: 再做一步应该就可以了
```

### 动态工具选择
```
Thought: 这是一个除法问题
Action: divide(10, 2)  # 选择正确的工具

Thought: 现在我需要做加法
Action: add(5, 3)  # 切换工具
```

## 研究起源

ReAct 由以下论文提出：
> **"ReAct: Synergizing Reasoning and Acting in Language Models"**
> Yao et al., 2022
> 论文链接：https://arxiv.org/abs/2210.03629

核心发现：将推理轨迹与特定任务的行动相结合，能创造出比单独使用任一方法都更强大的智能体。

## 为什么要学习这个模式？

### 1. 现代智能体的基础
几乎所有生产级智能体系统都使用 ReAct 或类似的模式。

### 2. 可理解的 AI

### 5. 生产就绪
该模式从演示到实际应用都能良好扩展。

## 总结

ReAct 将大语言模型（LLM）从：
- **脆弱的计算器** → 可靠的问题解决者
- **黑盒** → 透明的推理者
- **一次性回答者** → 迭代式思考者
- **孤立的模型** → 使用工具的智能体

它是连接语言模型与能够可靠完成复杂任务的自主智能体之间的桥梁。
