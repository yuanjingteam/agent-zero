# 提示工程（Prompt Engineering）

提示工程为塑造智能体行为提供了最快速、最直接的方法——定义其个性、功能和选择（例如何时应该使用工具）。智能体使用两种提示类别：系统级提示（System-level prompts）和用户级提示（User-level prompts）。

用户级提示由用户在对话中输入的消息组成。这些消息在每次交互时都会变化，且不受开发者的控制。

系统级提示包含开发者设定的指令，在整个对话过程中保持不变。这些指令定义了智能体的语气、能力、限制以及工具使用指南。

查看 Anthropic 的系统提示

https://docs.claude.com/en/release-notes/system-prompts#september-29-2025

## 提示设计（Prompt Design）

在为智能体创建提示时，你需要实现两个目标：

1. 让智能体很好地解决问题

   - 帮助它正确完成复杂任务
   - 实现清晰、逻辑性的思考
   - 减少错误

2. 保持智能体的个性一致性

   - 定义智能体是谁以及它如何说话
   - 匹配你的品牌声音
   - 根据每个情况以适当的情感回应

这两个目标同等重要。一个准确但粗鲁的回答会损害用户体验。一个友好但没有实际帮助的回答是无用的。

## 提示策略（Prompt Strategies）

### 智能体角色（Agent Role）

给大语言模型（LLM）一个特定角色可以改善其回复——它会自然地采用该角色的词汇和专业知识。

示例：

"你是一名儿科医生" → 使用医学术语，讨论儿童发育，推荐适合年龄的治疗方法
"你是一名厨师" → 解释烹饪技巧，建议食材替代品，讨论风味特征
"你是一名高中数学老师" → 分步骤分解问题，使用简单语言，提供练习示例
"你是一名初创公司创始人" → 关注增长，使用业务指标，考虑可扩展性

让角色具体化：
不要用："你是一名作家"
更好的是："你是一名科技博主，为初学者简化复杂的 AI 概念"

角色最适合用于专业问题，应设置在系统提示中。

### 具体明确，不要模糊

大语言模型（LLM）会按字面意思解释指令。模糊的提示会产生随机结果。具体的提示会产生一致的输出。

模糊与具体示例：

❌ 模糊："写一些关于狗的内容"
✅ 具体："写一个 3 段的指南，教小狗学会坐下"

❌ 模糊："让它更好"
✅ 具体："修复语法错误并缩短到 100 字以内"

❌ 模糊："专业一点"
✅ 具体："使用正式语言，避免缩写，称呼读者为'您'"

❌ 模糊："分析这些数据"
✅ 具体："找出前 3 个趋势并解释每个趋势的原因"

为什么这很重要：大语言模型有数千种方式来解释模糊的指令。它会猜测你想要什么——而且经常猜错。明确的指令消除了猜测，让你能够控制输出。

经验法则：如果一个人类助手需要提出澄清性问题，那你的提示就太模糊了。

### 使用 JSON 结构化 LLM 输入

使用 JSON 来结构化你的输入可以帮助大语言模型更清晰地理解任务，并使集成更容易。与其发送一大段文本，不如将你的请求分解为带标签的部分，如任务、输入、约束和输出格式。

优势
- 清晰性：JSON 键向模型展示每个部分的含义。
- 可靠性：更容易解析和验证响应。
- 一致性：减少随机或叙述性回答。
- 集成性：与 API 和模式配合良好。

最佳实践
- 保持简单和浅层——避免深层嵌套。
- 使用描述性键名（"task"、"context"、"constraints"）。
- 告诉模型确切的输出格式（例如，"仅以有效的 JSON 响应"）。
- 可选地定义 JSON 模式以强制结构。
- 始终在代码中验证响应。

示例
````
{
  "task": "summarize",
  "input_text": " - Article text here. - ",
  "constraints": {
    "max_words": 100,
    "audience": "non-technical"
  },
  "output_format": {
    "type": "JSON",
    "schema": {
      "summary": "string",
      "key_points": ["string"]
    }
  }
}
````

这种结构化格式帮助模型区分要做什么、使用什么数据以及如何回复，从而产生更一致、机器可读的输出。

### 少样本提示（Few-Shot Prompting）

少样本提示意味着在要求大语言模型执行新任务之前，给它几个你想要的示例。就像给学生展示两三个已解决的问题，让他们理解模式。

示例
```
Example 1:
Feedback: "The room was clean and quiet."
Category: Positive

Example 2:
Feedback: "The staff were rude and unhelpful."
Category: Negative

Example 3:
Feedback: "Breakfast was okay, but the coffee was cold."
Category: Neutral

Now categorize this:
Feedback: "The view from the balcony was amazing!"
Category:
```

模型从示例中学习并以相同的风格继续——在这里，它会回答："Positive"

当你想要一致的语气、格式或逻辑而无需重新训练模型时，少样本提示很有用。

### 思维链（Chain of Thought）

思维链意味着要求大语言模型逐步思考，而不是直接跳到答案。它帮助模型更好地推理，特别是对于逻辑、数学或多步骤问题。

示例

问题：如果 3 个苹果花费 6 美元，那么 5 个苹果花费多少？
让我们逐步思考。

模型推理：
3 个苹果 → 6 美元 → 每个苹果 2 美元。
5 个苹果 × 2 美元 = 10 美元。

答案：10 美元

通过鼓励逐步思考，你帮助模型减少错误并清晰地解释其推理过程。
