# 贡献指南

感谢您考虑为 AgentZero 做出贡献（contribute）！

## 项目理念

本仓库通过从零构建的方式教授 AI 代理（AI Agent）的基础知识。每项贡献都应服务于这一学习目标。

**核心原则：**
- **清晰优先于技巧** - 代码（code）应该易于理解
- **基础优先** - 没有黑盒或魔法
- **渐进式学习** - 每个示例（example）都建立在前一个基础上
- **本地优先** - 不依赖 API

## 贡献类型

### Bug 报告
发现了问题？请提交一个 Issue，包含以下信息：
- 哪个示例（`intro/`、`react-agent/` 等）
- 预期行为与实际行为
- 您的环境（Node 版本、操作系统、使用的模型）
- 重现步骤

### 文档改进
- 错别字和语法修正
- 更清晰的说明
- 更好的代码注释
- 文档中的额外示例
- 图表和可视化

### 新示例
想添加新的代理模式（pattern）？太好了！请：
1. **先提交一个 Issue** - 让我们讨论一下是否合适
2. 遵循现有结构：
- `pattern-name/pattern-name.js` - 可运行的代码
- `pattern-name/CODE.md` - 详细的代码详解（walkthrough）
- `pattern-name/CONCEPT.md` - 为什么重要、使用场景
3. 保持简洁并添加充分注释
4. 使用至少一个模型进行充分测试

### 代码改进
- 性能（performance）优化（附基准测试（benchmark））
- 更好的错误处理（error handling）
- 更清晰的变量（variable）名
- 更有用的控制台（console）输出

## 我们不接受的内容

- 框架（framework）集成（LangChain 等）- 本仓库教授的是它们的底层原理
- 云 API 示例 - 保持本地运行
- 生产环境功能（监控、扩展）- 这是教学项目
- 复杂的抽象 - 保持对初学者友好

## 贡献流程

1. **Fork** 仓库
2. **创建分支（branch）**：`git checkout -b fix/issue-description`
3. **修改代码**并充分测试
4. **提交（commit）**并附上清晰的信息：`git commit -m "Fix: clarify ReAct loop explanation"`
5. **推送（push）**：`git push origin fix/issue-description`
6. **提交拉取请求（Pull Request）**，包含：
- 清晰的标题
- 描述修改了什么以及原因
- 关联的 Issue（如果有）

## 代码规范

- 使用清晰、描述性的变量名
- 添加注释解释*为什么*，而不仅仅是*是什么*
- 遵循现有代码风格（没有代码检查工具（linter），只需匹配现有模式）
- 保持示例自包含（尽可能一个文件）
- 提交前使用 Qwen 或 Llama 模型进行测试

## 文档规范

- 使用清晰、简洁的语言
- 在代码之前解释概念
- 在有帮助的地方包含图表（ASCII 艺术也可以！）
- 提供真实世界的使用场景
- 链接到相关示例

## 示例结构
```
new-pattern/
├── new-pattern.js # 可运行的代码
├── CODE.md # 逐行详解
└── CONCEPT.md # 高层概念解释
```

**CODE.md 应包含：**
- 先决条件（prerequisites）
- 逐步代码分解
- 如何运行
- 预期输出

**CONCEPT.md 应包含：**
- 解决什么问题
- 为什么这个模式重要
- 真实世界的应用
- 简单的图表
